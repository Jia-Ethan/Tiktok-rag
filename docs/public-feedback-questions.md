# Public feedback questions

Use these questions if you want to collect early feedback on the project without forcing people into a long survey.

## Product direction

1. What short-video knowledge workflow would make this useful for you?
2. Is a local-file-first version already valuable, or do you only care once URL ingestion is reliable?

## Technical direction

3. After transcription, what should come first: chunking, retrieval, or better URL ingestion boundaries?
4. Would you rather see this stay as a developer toolchain, or gradually become a more user-facing product?

## Quality expectations

5. What level of transcript quality is “good enough” for your workflow?
6. Do you need timestamps mainly for retrieval, review, or jump-back-to-video playback?
