# Feedback guide

If you want to help shape `video-rag`, the most valuable feedback is workflow-specific feedback.

## What to include

- Where your video files come from
- Typical video duration
- Main language or language mix
- Whether the audio is clean, noisy, multi-speaker, or mixed with music
- What you want to do after transcription:
  - retrieval
  - summary
  - knowledge organization
  - prompt context construction

## Use Discussions when

- You want to talk about roadmap direction
- You want to describe a workflow or use case
- You want to explain what would make the project useful for you

## Use Issues when

- You found a reproducible bug
- You want to request a scoped improvement with a clear problem statement

## Useful framing

The most helpful reports usually answer:

1. What kind of video are you processing?
2. What part of the current output is already useful?
3. What is the first thing missing that blocks your next step?
4. If this repo improved by one step, what should that step be?
