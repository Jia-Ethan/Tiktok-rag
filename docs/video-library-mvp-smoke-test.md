# Video Library MVP smoke test

这份文档给出一组可复现的本地测试路径，用来验证“个人视频资料库 MVP”是否已经成立。

## 测试目标

在处理 3 个本地视频之后，确认你可以：

- 在视频库中看到 3 条资料
- 做筛选和排序
- 收藏、改显示标题、加标签、写备注
- 跨视频做关键词搜索
- 跳回具体视频详情页并定位时间段
- 在详情页继续做 grounded QA
- 在详情页看到历史保存结果

## 准备 3 个本地视频

你可以直接使用自己现有的 3 个本地视频。

为了更容易验证，建议：

- 三个视频都有人声
- 三个视频的主题或关键词有明显差异
- 至少有两个视频包含你能稳定搜索到的关键词

推荐示例：

- `video-1.mp4`：讲产品规划
- `video-2.mp4`：讲 AI / 模型 / prompt
- `video-3.mp4`：讲效率 / 工作流 / 知识整理

## 执行步骤

### 1. 启动应用

```bash
python3 app/gradio_app.py
```

### 2. 处理 3 个视频

在「处理新视频」里重复 3 次：

1. 选择本地视频
2. 保持模型为 `base`
3. 语言先用 `auto`
4. 点击“开始处理”

### 3. 验证视频库首页

切到「个人视频库」，确认：

- 能看到 3 条视频资料
- 每条资料都有标题、时间、时长、语言、状态、摘要预览

### 4. 验证筛选 / 排序

依次测试：

- 标题关键词筛选
- 语言筛选
- 收藏状态筛选
- 最近处理排序

### 5. 验证整理能力

任选一个视频，确认你可以：

- 收藏
- 把显示标题改成更易认的名字
- 添加 1 到 2 个标签
- 写一段备注
- 保存后刷新页面，信息仍然存在

### 6. 验证跨视频关键词搜索

在「跨视频关键词搜索」里输入一个你知道会命中的词。

确认：

- 结果里显示命中的视频标题
- 结果里显示 chunk 编号
- 结果里显示开始 / 结束时间段
- 点击结果后，会自动进入对应视频详情并定位到对应 chunk

### 7. 验证详情页继续使用

在已经选中的视频详情页中，继续测试：

- 当前视频关键词搜索
- grounded QA
- 点击引用后定位到对应 chunk

### 8. 验证保存结果历史

在详情页中分别执行：

- 保存一次搜索结果
- 保存一次问答结果
- 保存一次单视频摘要

确认：

- 文件写入 `data/exports/<video_id>/`
- 详情页里出现“已保存结果”记录
- 重启应用后，保存结果历史仍然可见

## 最终验收

如果下面这些都成立，就说明这轮 MVP 基本跑通：

- 你不需要记 `job_id`，也能从视频库中找回视频
- 你可以跨视频搜索，并回到具体视频和时间段
- 你可以对视频做最基本的整理
- 你可以在详情页继续使用单视频搜索和 grounded QA
- 保存结果历史会留在本地视频资料里
